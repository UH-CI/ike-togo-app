After saving your metadata, if you go back to your "My Data" page and look in the "status" column, you will see the word "annotated" there.




Marking files as "ready for group dissemination" is called "Staging." This process lets the system administrators know that you have added the necessary metadata and ensured your file is ready for group use. The administrator will verify this and then publish your file to the repository. To stage multiple files with one action, all files all need to be in the same directory. The process is the same regardless of if you are affecting just one file or multiples. First, go to the My Data page, then check the box on the left side next to each file you want to stage. At the top of the files section of the page, click the "Actions" menu to see the following.



Click the "Stage/Unstage For Repository" menu item. When the page refreshes, all affected files should now show as "staged" in the status column. Once the process completes, you should see the status column for each affected file change to include the word "Staged."



When each file is approved by the system administrator, the status will change to "published." Should you later change your mind and decide the file is not ready for staging, follow the exact same process to reverse the status and unstage the file.

The process to un-stage a file is exactly the same as the one you followed to stage it originally. To unstage multiple files with one action, all files all need to be in the same directory. The process is the same regardless of if you are affecting just one file or multiples. First, go to the My Data page, then check the box on the left side next to each file you want to un-stage. At the top of the files section of the page, click the "Actions" menu to see the following.



Click the "Stage/Unstage For Repository" menu item. When the page refreshes, all affected files' status column should now be empty.
